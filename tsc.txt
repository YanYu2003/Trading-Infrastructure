五、推荐项目阶段

请你以以下阶段为基础进一步优化：

Phase 0：项目设计与知识讲解

* 输出架构图的文字版说明；
* 输出模块划分；
* 输出数据流；
* 输出最小 MVP；

* 如适合，创建 AGENTS.md，写明项目开发规范、测试要求、提交规范、禁止真实资金交易等规则。

Phase 1：核心数据模型与订单状态机

* 定义 Symbol / MarketDataEvent / Quote / Trade；
* 定义 Order / OrderSide / OrderType / OrderStatus；
* 定义 Fill / Position / Account / PnL；
* 实现订单状态机；
* 写单元测试。

Phase 2：Mini OMS + Risk Engine + Mock Broker

* 实现 OrderManager；
* 实现 RiskEngine；
* 实现 MockBroker；
* 支持 market order 和 limit order；
* 支持 created、submitted、partially_filled、filled、cancelled、rejected 等状态；
* 支持持仓、现金、PnL 更新；
* 支持简单风控规则：

  * 单笔最大下单金额；
  * 单只股票最大持仓；
  * 现金余额检查；
  * 重复订单检查；
  * kill switch；
* 写单元测试和集成测试。

Phase 3：行情接入与事件驱动链路

* 先实现 MockMarketDataProvider；
* 再考虑接入公开行情 API 或 Alpaca Market Data；
* 支持 WebSocket 或模拟 WebSocket 事件流；
* 形成：
  market data event -> strategy signal -> risk check -> order -> broker -> fill -> position update
  的完整闭环；
* 增加日志和异常处理；
* 写集成测试。

Phase 4：简单策略与回放

* 实现一个非常简单的策略，例如 moving average crossover 或 price threshold strategy；
* 不追求收益，只追求系统链路完整；
* 支持历史行情回放；
* 输出交易记录、持仓变化、PnL 变化；
* 生成简单报告。

Phase 5：Paper Trading Adapter，可选

* 在 Mini OMS 和 Mock Broker 已经稳定后，再接入 Alpaca Paper Trading；
* 不要一开始就依赖第三方平台；
* 使用环境变量保存 API key，不要硬编码；
* 严格区分 paper endpoint 和 live endpoint；
* 默认禁用 live trading；
* 如果无法注册或无法接入 Alpaca，请保留 Mock Broker 作为完整可运行方案。

Phase 6：工程化与简历化

* 增加 README 项目说明；
* 增加架构图说明；
* 增加测试覆盖说明；
* 增加运行示例；
* 增加 demo 数据；
* 增加性能指标，例如订单处理耗时、行情事件处理吞吐量、异常重连次数；
* 最后帮我生成一版适合写进中文简历的项目描述和一版英文简历项目描述。
